# 单细胞跨模态翻译算法设计思路

## 1. 核心设计思想

本项目没有采用直接端到端回归的粗暴方案，而是利用一个重要生物学先验：

> 同一细胞类型的 RNA 表达通常具有较强相似性。

因此，项目将复杂的跨模态预测任务拆成两个子任务：

1. **软分类预测**：先通过甲基化数据推断细胞属于各 MajorType 的概率；
2. **残差修正**：再用深度神经网络捕捉细胞类型原型无法解释的个体差异。

这种“先抓大局、再抠细节”的策略可以降低模型学习难度，并提高可解释性。

## 2. 算法流程

### 2.1 预处理与降维

对甲基化矩阵进行：

1. 缺失值填补；
2. 零方差特征删除；
3. 标准化；
4. PCA 降维。

PCA 将高维甲基化特征压缩到低维空间，在保留主要判别信息的同时降低过拟合风险。

### 2.2 细胞类型软分类

使用逻辑回归在 PCA 后的甲基化特征上预测 `MajorType`。关键是使用：

```python
predict_proba
```

而不是硬分类 `predict`。这样每个细胞都可以用“属于各类型的概率向量”表示，保留细胞状态的连续性。

### 2.3 原型表达构建

按 `MajorType` 分组计算训练集中每种类型的平均 RNA 表达，即“原型表达”。测试集初步预测为：

\[
\widehat{Y}^{proto}_{test}=P_{test}C
\]

其中 \(P_{test}\) 是测试集类型概率矩阵，\(C\) 是类型原型表达矩阵。

### 2.4 残差学习

训练集残差定义为：

\[
R_{train}=Y_{train}-\widehat{Y}^{proto}_{train}
\]

然后训练残差神经网络学习：

\[
g_{\theta}(Z_{train}) \approx R_{train}
\]

网络使用残差块和 BatchNorm，以增强深层网络的训练稳定性。

### 2.5 最终融合与后处理

最终预测为：

\[
\widehat{Y}_{test}
=
\widehat{Y}^{proto}_{test}
+
\alpha \cdot g_{\theta}(Z_{test})
\]

其中 \(\alpha\) 是残差权重，默认值为 0.4。使用小于 1 的权重是一种保守策略，可以减少残差噪声对测试集预测的放大。

最后将预测值中的负数截断为 0，以符合 RNA 表达非负的基本约束。

## 3. 方案优势

该方案将“跨模态翻译”转化为：

```text
软分类 → 原型表达 → 残差修正
```

每个模块职责清晰：

- 原型表达捕捉细胞类型主导的共性表达；
- 残差网络捕捉细胞内部状态和染色质状态差异；
- 残差权重控制模型复杂度和噪声放大。

## 4. 局限性

1. 当前实现仍将矩阵转为 dense array，大数据集上可能占用较多内存；
2. 逻辑回归只利用 MajorType 标签，未显式建模更细粒度细胞状态；
3. 残差权重需要更系统的验证集搜索；
4. 未引入 gene regulatory network 等更强生物学先验。

## 5. 后续改进方向

- 引入自编码器或对比学习构建共享潜空间；
- 使用图神经网络建模细胞邻域关系；
- 对不同基因分组建模；
- 使用更严格的验证集划分评估泛化能力；
- 加入更细粒度的细胞类型或发育阶段信息。
